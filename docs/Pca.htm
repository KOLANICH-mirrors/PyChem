Principal Components Analysis

In the analysis of high-dimensional data where many tens of thousands of array elements are generated, it is often necessary to reduce the number of variables prior to further analysis.  Principal components analysis (PCA) is an unsupervised method of data reduction where the original data matrix is projected on to a smaller variable subspace.  With PCA the first few principal components (PCs) typically account for a majority of the variance in the data.  Useful references are listed in the Introduction.

Running PCA

To run PCA select either the Raw, or Processed spectra (if applicable) from the dropdown.  Selecting Correlation matrix mean centres the data prior to PCA, the Covariance option column normalises (autoscales) the data.  PCA can be run using either the singular value decomposition (SVD) or non-iterative partial least squares (NIPALS).  SVD extracts the maximum number of PCs from the data, where as NIPALS extracts a user defined number iteratively.  The choice of which algorithm to use is not critical.  Finally, to run PCA, press the  icon .

 

Exporting Results

Pressing the Export icon  allows the results to be saved to a plain text file.
