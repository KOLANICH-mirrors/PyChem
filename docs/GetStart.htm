Getting Started
  1. Import a 2D numeric array of data using the File...Import... option.  This dialog allows data to be loaded in to PyChem, the format must be for samples (dependent variables) to be ordered as rows of the array and measured variables (independent variables) as columns.  In the example below, the column and row headings would not be included in the file to be imported.  In the event that the data to be imported is ordered with samples as columns and measured variables as rows then the transpose option must be applied.
     

 

  1. In the Experiment Setup screen (see Experiment Setup), as a minimum a vector of Labels must be provided for each sample.  For any supervised analysis such as PLSR or DFA a Class structure is also required and this must consist of a consecutive series of integers from 1 to n that defines a priori groups within the data, such as machine, biological or experimental replicates.  In addition, for the purpose of cross-validation, each sample must be identified as one of a Train, Test or Validation replicate (there is an option to randomly assign these identifiers). 

These values can be pasted from an Excel spreadsheet, or similar by right clicking on the first cell in the range to be pasted and selecting the Paste option from the pop-up menu (see General Functionality).  Multiple columns can be generated to define different relationships within the data.

Alternatively, this information can be uploaded in the .csv (comma separated file format) which can be created using the "Save as..." option in Microsoft Excel.  The format requires that the column labels are provided, along with a value in each column for every row (or sample) in the experimental data set.  Examples are distributed with the application and can be found in /PyChem/examples.

  1. With the data loaded in to PyChem, and some of the metadata entered in to the Experiment Setup grid, you are ready to run some analysis or save the work so far as a PyChem Experiment (see Experiment Manager).
