A genetic algorithm coupled to partial least squares calibration (GA-PLSC)

This genetic algorithm can be used for feature selection, where a quantitative PLS model has been formulated and the user is interested in understanding the variables that are most important for the model.  The GA directs a search to formulate a model with the lowest possible value for the model validation error.


Running the GA

Common to the three GA feature selection applications available with PyChem are the following options:

No. vars from/to: positive integers, for selecting the variable subset range over which the GA should search

No. inds: positive integer, the number of individuals to use in the GA population

No. runs: positive integer, the number of independent GA runs

Xover rate: real value between 0 & 1, the probability of crossover (here only single point crossover is used)

Mut. rate: real value between 0 & 1, the probability that a gene will mutate

Insert. rate: real value between 0 & 1, the number of individuals selected for evolution at each generation

Max. factors: positive integer, the maximum number of latent variables to extract, for the PLS algorithm this will always have a maximum of [No_Vars-1]

Max. gen: positive integer, the total number of generations for each independent GA run

Rep. until: positive integer, an alternative to Max. gen, where the GA terminates after the fittest solution has not changed for the number of generations specified

Interpretation of Results

To view a model, simply double click on an entry in the results tree.  The RMS Error Comparison graph plots the cross-validation error for models formulated across the range of variable subset lengths.  This gives an indication of how well the models are performing when given different levels of information.  It can be useful for seeing where model improvement begins to plateau regardless of the amount of new information given to the model. 

The GA Optimisation plot simply show the minimum fitness achieved by the GA at each generation.

The Frequency of Variable Selection plot shows the frequency with which individual variables are selected across the total number of independent GA runs.   This gives an indication of which variables or regions are important for the model.  Finally, the formulated model is also plotted.

PLS Model shows the actual calibration curve based on the selected variable subset.

LS Results lists the ordinary least squares results for the training, cross validation and test samples.

Model Error shows the training and cross-validation error curves

Spectral Loadings shows the loadings plots for each of the extracted latent variables

Feature Plot is a scatter plot of the original spectral variables within the variable subset

Exporting GA Results

Press the Export button to bring up the Export Results dialog.  Select a folder to save the files by pressing the Browse... button.  The specific output that is required can be customised by using the checkboxes.

The files will be saved as a tab delimited ASCII file to the selected directory under chroms.txt, scores.txt and curves.txt.

chroms.txt - the variables are encoded 1 : n (where n is the total number of independent variables or measured x-variables).  Any zeros are for padding only and can be ignored.

scores.txt - just a column vector of objective scores which relate directly to the variable subset specified in each row of chroms.txt.

curves.txt - the optimisation curve relating to each model in chroms.txt.
